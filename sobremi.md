### Descripción corta

DocuMente es un asistente fullstack que permite cargar documentos PDF/TXT, generar embeddings, recuperar chunks relevantes por similitud coseno y responder en lenguaje natural con fuentes citables.

### Descripción extendida

DocuMente demuestra una arquitectura fullstack con TypeScript para trabajar con documentos y preguntas en lenguaje natural. El usuario puede subir documentos, el backend procesa el texto, lo divide en chunks con overlap, genera embeddings con OpenAI, guarda los chunks en SQLite y luego recupera los fragmentos más relevantes para responder preguntas con contexto y fuentes. También incluye un RAGAgent con skills atómicas para búsqueda, generación de respuesta, resumen y extracción de keywords.

### Flujo RAG

1. El usuario sube un PDF o TXT.
2. El backend valida JWT, tipo MIME y tamaño.
3. El archivo se procesa en memoria.
4. Se extrae y sanitiza texto.
5. El texto se divide en chunks con overlap.
6. OpenAI genera embeddings con `text-embedding-3-small`.
7. SQLite guarda documentos, chunks y embeddings.
8. En chat, el agente genera embedding de la pregunta.
9. Se calcula similitud coseno contra los chunks.
10. Se filtran y ordenan chunks relevantes.
11. El sistema responde con contexto y fuentes.

### Arquitectura backend

- `routes`: endpoints HTTP sin lógica de negocio.
- `controllers`: orquestan request/response.
- `services`: contienen lógica de negocio, RAG, auth y documentos.
- `services/skills`: skills atómicas del agente RAG.
- `middleware`: JWT, upload, rate limit y validación.
- `db`: conexión SQLite y schema.
- `validators`: schemas Zod.
- `utils`: chunking, similitud, serialización y logs.
- `types`: contratos TypeScript.

### Arquitectura frontend

- `components`: UI, auth, documentos y chat.
- `pages`: login, registro, dashboard, chat y settings.
- `hooks`: `useAuth`, `useDocuments`, `useChat`.
- `store`: stores Zustand.
- `services`: clientes HTTP.
- `types`: contratos de API.
- `utils`: constantes, errores y formatters.

### Nombre

### Estado

En desarrollo activo.

### Tipo de proyecto

Agente de QA automatizado con IA para convertir requerimientos funcionales en pruebas ejecutables.

### Descripción corta

QAPilot es una plataforma open source que convierte requerimientos funcionales en lenguaje natural en casos de prueba, ejecuta pruebas UI y API, analiza fallos con IA y genera reportes HTML, PDF y JSON.

### Descripción extendida

QAPilot está diseñado para ayudar a equipos QA y desarrolladores a pasar de requerimientos escritos en lenguaje natural a pruebas automatizadas. El sistema genera casos de prueba, crea pruebas UI con Playwright, ejecuta pruebas de APIs REST con Fetch/Axios, analiza fallos con IA y produce reportes exportables. El enfoque es local-first: la API key es del usuario y los datos corren localmente.

### Stack

- React.
- TypeScript.
- Node.js.
- OpenAI API.
- Playwright.
- SQLite.
- Docker.
- SSE.
- Fetch/Axios.
- HTML/PDF/JSON reports.
- AES-256-GCM.
- Helmet.
- CORS restrictivo.
- Zod.
- GitHub Actions.

### QAAgent

QAPilot usa un QAAgent con 7 skills atómicas:

- `analyzeRequirement`.
- `generateTestCases`.
- `generateUITests`.
- `generateAPITests`.
- `executeTests`.
- `analyzeFailures`.
- `generateReport`.

### Flujo general

1. El usuario escribe o carga un requerimiento funcional.
2. El agente analiza el requerimiento.
3. Se generan casos de prueba.
4. Se generan pruebas UI con Playwright.
5. Se generan o ejecutan pruebas API con Fetch/Axios.
6. Se ejecutan las pruebas.
7. Si hay fallos, la IA analiza los errores.
8. Se genera un reporte HTML, PDF y JSON.
9. El progreso se comunica en tiempo real vía SSE.

### Seguridad

- API key encriptada con AES-256-GCM.
- Prevención SSRF.
- Helmet.
- CORS restrictivo.
- Zod en todos los inputs.
- Queries parametrizadas.
- DSL declarativa segura.
- No ejecuta código arbitrario generado por IA.
- CI/CD con typecheck, lint, test y build.
- Dockerizado con docker-compose.

### Qué demuestra este proyecto

- Experiencia real de Kendall en QA Automation.
- Capacidad para convertir conocimiento QA en producto.
- Integración de IA con testing.
- Automatización UI con Playwright.
- Automatización API con Fetch/Axios.
- Diseño de agentes con skills atómicas.
- Seguridad aplicada a herramientas con IA.

- Reportería profesional.

### Respuesta recomendada
